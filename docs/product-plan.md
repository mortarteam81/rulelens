# 규정 신구조문 대비표 분석 대시보드 제품 계획

## 목적
성신여자대학교 규정 개정 실무에서 기존 공개 규정과 향후 개정안을 함께 비교하여, 조문별 변경사항·실무 영향·위험도·검토의견 초안을 빠르게 확인하는 웹 대시보드를 만든다.

## 핵심 사용 시나리오

### 1. 기존 공개 규정/개정이력 확인
- 사용자가 성신여대 규정관리시스템 URL 또는 규정명을 입력한다.
- 시스템이 공개 규정관리시스템에서 현행 규정, 개정 이력, 기존 신구조문 대비표를 가져온다.
- 기존 개정 패턴과 조문 구조를 기준 데이터로 사용한다.

### 2. 앞으로 변경할 규정개정안 검토
- 사용자가 HWP/PDF 개정안을 업로드한다.
- 개정안이 신구조문 대비표이면 구 조문/신 조문/개정 사유를 추출한다.
- 개정안 본문만 있으면 공개 현행 규정과 자동 비교한다.

### 3. 실무 검토 대시보드
- 조문별 변경 유형, 위험도, 실무 영향, 확인 질문을 표시한다.
- Korean Law MCP로 상위법령 근거를 확인한다.
- kordoc MCP로 HWP/HWPX/PDF 문서 파싱, 표 추출, 문서 비교를 보강한다.
- 근거가 없는 법령 판단은 “추가 확인 필요”로 표시한다.

## 입력 소스
1. 성신여대 규정관리시스템 공개 페이지
   - 최신 제·개정 목록
   - 현행 규정 상세
   - 신구조문 대비표 HTML
   - HWP/PDF 다운로드 링크
2. 사용자 업로드 파일
   - HWP
   - HWPX
   - PDF
3. 수동 입력
   - 규정명
   - 조문 텍스트
   - 관련 법령 키워드

## MVP 범위
- 성신 규정 URL 입력
- 공개 신구조문 대비표 HTML 수집/정규화
- HWP/PDF 업로드 parser boundary
- 공통 `ParsedComparisonTable` 구조
- 조문별 위험도/검토의견 표시
- parser confidence/warnings 표시
- MCP 연동 준비 인터페이스

## MVP 제외 범위
- OCR 기반 스캔 PDF 파싱
- 관리자서비스 `/process/*` 접근
- 대량 크롤링
- 자동 법률 자문 또는 확정 판단
- 공개 배포 전 익명 업로드

## 성공 기준
- 성신 규정 URL에서 기존 신구조문 대비표를 가져와 조문별로 표시한다.
- 업로드 파일도 동일한 파이프라인으로 들어갈 수 있는 구조를 가진다.
- 분석 결과에는 위험도, 변경 유형, 실무 영향, 확인 질문, 검토의견 초안이 포함된다.
- 빌드가 성공한다.
